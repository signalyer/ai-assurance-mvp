Target Architecture & Tech Stack · v1.0

The governance substrate for enterprise AI

A six-layer control plane that wraps every LLM call in three enforced decorators (@scrub_pii → @trace_llm_call → @evaluate_response), backed by a four-tier memory model, fed by commodity evaluation tools (Langfuse + DeepEval + Presidio), and governed by an organizational layer (risk inventory, governance body, RACI, regulatory posture). It is not an eval tool, not a tracing tool, not a guardrails tool — it is the control plane that makes those tools auditable, attestable, and defensible across a portfolio of AI systems in regulated industries.

01 Architectural tenets

Ten rules that govern every decision. If a feature violates any of these, it does not ship.

TenetWhat it means in practice
Compose, don't rebuild commoditiesWrap Langfuse + DeepEval + Presidio rather than build custom tracing / metrics / PII detection. Defensibility lives in the layers around them, not in reimplementing them.
PII never crosses the LLM boundary rawscrubber.tokenise_payload() runs BEFORE every tracer.trace_call(). Langfuse and any external LLM receive scrubbed payloads only. Verified at every call site.
Policy engine fails closedOPA error → DENY. Engine unreachable → DENY. Better to block a release than to ship an unguarded one.
No SaaS guardrails in the prompt pathAll guardrail tools (Presidio, NeMo Guardrails, Llama Guard 3) self-hosted. No external prompt routing.

Headless engine + thin clientsEngine is API-only. UIs (Portal, Console) and CLI are skins. OpenAPI contract drives codegen for all clients.
Compound engineeringEvery mistake corrected becomes a date-stamped rule in CLAUDE.md. The platform learns from its own history.

02b Deployment shape — headless engine + thin clients

The three rules that make this work

  1. OpenAPI is the contract. The engine emits an OpenAPI spec. SPAs and SDK are generated against it. No drift possible — contract tests in CI fail the build if the API breaks compatibility.
  2. The event log is the source of truth. Engine writes to events.jsonl first, then projects to Postgres. Materialized views are derived. Any view can be rebuilt by replaying the log.
  3. UIs render API responses. Zero business logic in the SPA layer. SPAs are ~3–5K lines each, share a component library, and have no local state beyond the current request.

03 Three-decorator chain

The runtime invariant. Every LLM call in the platform passes through these three decorators in this exact order. Enforced at SDK level — using @trace without @scrub_pii raises MissingScrubberError at decorator application time.

from signallayer import scrub_pii, trace, evaluate

# Production agent code — three lines wire the full L3 + L5 stack

@scrub_pii
@trace(workload="payments-exception")
@evaluate(pack="rag-default-v1")
async def handle_dispute(query: str, context: dict) -> str:
    ...

What each decorator does

OrderDecoratorResponsibilitiesBacked by
1 @scrub_pii Detect PII / IP / secrets in args and return value · tokenise via Presidio NER + 12 regex patterns · write reversible token map to Fernet-encrypted vault · pass scrubbed payload downstream L3 · scrubber.py · domain/deid_vault.py
2 @trace_llm_call Open Langfuse trace · capture scrubbed prompt + scrubbed response + model + tokens + latency · attach workload ID + agent ID + system ID + provider · emit llm.call.completed event L5 · tracing/langfuse_client.py
3 @evaluate_response Run configured eval pack (Tier A mandatory + Tier B risk + Tier C team) · execute 6 DeepEval scorers in parallel · attach scores to trace · compute 3-state classification per scorer · emit eval.run.completed event · trigger policy gates if blocking L5 · eval/runner.py · DeepEval 6-metric
Today's critical defect tracer.py sends raw prompts to Langfuse. The scrubber is not yet inserted in the chain. This is the Day 1 Hour 1 emergency fix. Until patched, no enterprise demo can run safely.

04 Four-tier memory model

Memory is not just RAG. RAG is one tier of four. Each tier has its own retention policy, access control, audit, right-to-forget semantics, and PII boundary.

TierTime horizonBacking storePII riskMitigation
T1 · In-context This call only LLM context window HIGH — user turns @scrub_pii before LLM sees it
T2 · Episodic Days–weeks per session JSONL on disk · data/episodes_{workload_id}.jsonl HIGH — episodes contain raw user turns Write-time scrubbing · episodes stored already-scrubbed · token vault holds originals
T3 · Semantic / RAG Months–years · org-wide Azure AI Search · embedded chunks · pinned to text-embedding-3-small CATASTROPHIC — indexed PII is searchable forever Index-time scrubbing · index_document() rejects docs with HIGH-confidence PII unless classification permits
T4 · Procedural Static · owned by platform domains.py · control libraries · decision rules None — no customer data n/a

async def build_context(workload_id, session_id, query, token_budget=8000):
    procedural = domains.get_procedural(workload.domain)               # T4
    rag_chunks = await rag_engine.retrieve_chunks(workload, query)  # T3
    episodes   = await selective_recall(workload, query, k=5)        # T2

    # Assemble T1 within token budget — system + T4 + T3 + T2 + query
    return assemble(procedural, rag_chunks, episodes, query, token_budget)


# Write path after every turn

async def write_episode(session_id, role, content, tool_calls=None):
    scrubbed, vault_ref = scrubber.tokenise_payload(content, scope=session_id)
    storage._append_jsonl("episodes_{wid}.jsonl", {
        "episode_id": uuid(),
        "session_id": session_id,
        "role": role,
        "content_scrubbed": scrubbed,
        "scrub_token_map_ref": vault_ref,
        "tool_calls": tool_calls,
        "created_at": now(),
    })

Compression + recall

  • compress_episode(session_id) — triggered when token count > 4000. Uses Claude Haiku to summarise. Original episodes marked compressed_at, summary written as new episode with provenance link.
  • selective_recall(query, k, tier_filter) — embedding-similarity search over T2 episodes + T3 chunks. Deduplicated by source. Ranked by relevance × recency × classification accessibility.

Index-time scrubbing for RAG

Tier 3 (Azure AI Search) is the highest-risk store. PII indexed there is searchable forever. rag_engine.index_document() enforces:

  • Scrub every chunk before embedding
  • If HIGH-confidence PII detected and document classification is not public with explicit allow → reject document + audit log entry
  • Record scrub_audit_hash (SHA-256 of pre-scrub document) on every chunk → proves a scrubbed version was indexed
  • Pin embedding model version in index name (aigovern-rag-{workload}-embsmall-v1) — upgrades are explicit, not silent

Six evaluation contexts

  1. Pre-LLM call — @policy_gate decorator queries OPA with system + agent + scrubbed prompt metadata + user + tool intent
  2. Release gate evaluation — each gate is a Rego rule against eval results + controls + findings
  3. Control applicability — at intake + reassessment · which controls fire for this classification + posture
  4. Agent binding authorization — when a system subscribes to a reusable agent · classification + version checks
  5. Tool authorization — runtime · per tool call · agent + tool + user context
  6. Right-to-forget cascade — which stores to purge for a subject

Deployment

Each AI System (deployable unit) has 1..N agents. Agents are versioned and reusable across teams. Publishing v2 of a reusable agent triggers re-evaluation for every subscriber.

Publish / subscribe semantics

  • Team Risk publishes v2 of pii-scrubbing-utility-agent
  • Engine emits agent.version.published event with diff vs previous version
  • Every subscriber (Team Payments, Team CX) receives a notification + re-evaluation window (7 days default)
  • Subscriber can: pin v1 (no upgrade) · accept v2 (re-evaluate with new eval run) · fork (custom agent)
  • After expiration: auto-pin enforced + finding opened "subscriber failed to re-evaluate"

Cross-agent gates (System-level)

Multi-agent systems carry release gates evaluated against the agent graph as a whole:

  • Orchestration correctness — does the agent graph reach a valid terminal state on representative inputs?
  • Plan coherence — do agent-to-agent handoffs preserve task intent? (LLM judge with calibration set)
  • Authority boundary — does any sub-agent escalate beyond its role's RBAC?
  • Weakest-link rule — the system's risk tier inherits the highest tier among its agents

11 Right-to-forget cascade

A single request from a regulator cascades across every store where the subject's data could exist. Verification report with SHA-256 hashes proves each cascade step completed.

async def forget(subject_id: str, scope: Literal["global", "system"], target_id: str | None):
    # Cascade in dependency order

    # 1. Vault — revoke token map entries
    v = vault.forget_by_subject(subject_id)
    emit("forget.vault.complete", removed=v.count, hash=v.verification_hash)

    # 2. Tier 2 episodic — secure delete
    e = episodic.forget_by_subject(subject_id, scope, target_id)
    emit("forget.tier2.complete", removed=e.count, hash=e.verification_hash)

    # 3. Tier 3 RAG — delete chunks by source_id
    r = rag.forget_by_subject(subject_id, scope, target_id)
    emit("forget.tier3.complete", removed=r.count, hash=r.verification_hash)

    # 4. Langfuse — delete traces via API
    l = langfuse.delete_traces_by_metadata(subject_id=subject_id)
    emit("forget.langfuse.complete", removed=l.count)

    # 5. Audit log entry with verification bundle
    bundle = {
        "subject_id": subject_id,
        "requested_by": actor,
        "approved_by": approver,
        "cascade_results": [v, e, r, l],
        "composite_hash": sha256(v.hash + e.hash + r.hash + l.count),
    }
    storage._append_jsonl("data/forget_audit.jsonl", bundle)
    return bundle

15 Data flow paths

Every operation in the platform flows through one of three paths.

Path A · LLM call (the hot path)

User input arrives at an agent
    │
    ▼

    ├─ Tier 2 episodes (selective_recall)
    └─ Assembles Tier 1 within token budget
    │
    ▼
@scrub_pii(args)
    ├─ Presidio entity detection
    ├─ token vault writes for any PII found
    └─ returns scrubbed prompt
    │
    ▼
@trace_llm_call → Langfuse.trace_call(scrubbed_prompt)
    │
    ▼
LLM abstraction (Claude / GPT-4o-mini / Bedrock)
    └─ provider routing via assurance_providers per workload sensitivity
    │
    ▼
@scrub_pii(response) on the way back
    │
    ▼
@evaluate_response → DeepEval scores attached to trace
    ├─ Tier A: PII leakage · hallucination · toxicity · scrub score
    ├─ Tier B: faithfulness · injection resistance (if HIGH/CRIT)
    ├─ Tier C: team-discretionary scorers
    └─ policy gate check → block / warn / pass
    │
    ▼
agent_memory.write_episode(session, turn) → Tier 2 append (scrubbed)
    │
    ▼
Response returned to user

Document arrives for indexing
    │
    ▼

    ▼

    │
